Functions

Functions are the basic foundations of R. From opening datasets to produce complex models and plots.

Functions have an argument list that can be empty (default expression terms) or have multiple expression terms.

# print character strings
print("Hello world!")
## [1] "Hello world!"
# join character strings
paste("Hello", "world","!")
## [1] "Hello world !"
# join character strings (multiple expression terms)
paste("Hello", "world","!",sep="-")
## [1] "Hello-world-!"
# determine the mean of a vector
myData <- c(2,4,3,2,3,4,2,2)
mean(myData)
## [1] 2.75
# find the arguments, needs and outputs for each function
?mean

# determine the standard deviation of a vector
sd(myData)
## [1] 0.8864053
# random numbers drawn from a given distribution without replacement 
sample(1:30,size=5) 
## [1] 28 23 25  9 24

sample(1:10,size=5, replace = FALSE) 
## [1] 10  3  9  4  5
Writing functions

It is useful to write your own functions to facilitate recurring computations and improves clarity of scripts. Functions result in:

  1. more clearly describe and isolate a particular task in your data analysis workflow.
  2. re-use code, rather than repeating the same steps several times. This should lead to faster development of scripts and to fewer mistakes. And if there is a mistake it only needs to be fixed in one place.

If the end of a function is reached without calling return, the value of the last evaluated expression is returned.

# function without arguments
greetings <- function(){
  print('Hello world')
    }
greetings()
## [1] "Hello world"
# function with one argument
greetings <- function(name){
    print(paste('Hello',name))
    }
greetings('students')
## [1] "Hello students"
greetings('professor')
## [1] "Hello professor"
# function with multiple arguments
myCalc <- function(data1,data2){
  result <- data1 / data2
  return(result)
    }
myCalc(4,2)
## [1] 2
# function with multiple arguments and levels
sumsquare <- function(a, b) {
    intermediateResult <- a + b
    result <- intermediateResult * intermediateResult
    return(result)
}
sumsquare(1,2)
## [1] 9
# functions can be vectorized (each argument can have more than one element)
x <- 1:3
y <- 2
sumsquare(x,y)
## [1]  9 16 25

Random numbers

It is common to create a vector of random numbers in data analysis, and also to create example data to demonstrate how a procedure works.

#
r <- sample(1:10,5)
r
## [1] 10  3  5  2  1
r <- sample(1:10,5, replace = FALSE)
r
## [1] 9 8 5 1 6

If you run the functions above, you will get different numbers. After all, they are random numbers! Well, computer generated numbers that are not truly random, but ‘pseudo-random’. To be able to exactly reproduce examples or data analysis we often want to assure that we take exactly the same ‘random’ sample each time we run our code.

To do that we use set.seed. This function initializes the random number generator to a specific point. This is a very important feature, as it allows us to exactly reproduce results that involve random sampling.

# random numbers within a given distribution
runif(5, min = 0, max = 1)
## [1] 0.9316087 0.7499363 0.1772600 0.7655666 0.3168787
runif(5, min = 0, max = 1)
## [1] 0.14997320 0.22946392 0.46326177 0.03568206 0.27111361
set.seed(12)
runif(5, min = 0, max = 1)
## [1] 0.06936092 0.81777520 0.94262173 0.26938188 0.16934812
set.seed(12)
runif(5, min = 0, max = 1)
## [1] 0.06936092 0.81777520 0.94262173 0.26938188 0.16934812
Control flows

Most programs have two general control-flow features: alternation and iteration. Alteration via if-then-else branches and iteration is done via loops.

Alternation

Alternation conditions are the most important building blocks in programming because they help structuring the logic of the code. We use conditional expressions if specific computations and actions should only be performed under certain conditions.

R knows three conditional expressions:

  1. if(condition){ command block }
  2. if(condition){ command block } else { alternative command block }
  3. ifelse(condition, command block, alternative command block)

These functions always require a Boolean condition, meaning that it needs to evaluate to TRUE or FALSE. Conditional expressions may be nested.

myVector <- c(1,4,3,2,6,5,4,6,7,6,5,22,3)

if( 4 %in% myVector ) { print("The vector contains the element 4") }

if( 42 %in% myVector ) { print("The vector contains the element 42") } else { print("The vector does not contain the element 42") }

ifelse( 4 %in% myVector , "Contains" , "Does not Contain" )

myConditionResult <- ifelse( 4 %in% myVector , 1 , 2 )
myConditionResult + 100

Iteration

Iteration via loops are crucial elements in programming. We use them, for example, for iterating (parts of) programs with different parameters or initial values.

R offers three different loop structures: repeat, while and for, the latter probably being the most commonly used.

The repeat loop is the most simplest but maybe also the most dangerous loop structure. If you do not set an appropriate break, the loop will iterate forever.

i <- 0 
repeat{
  i <- i + 1 
  print(i) 
  if(i>10) break
  }
## [1] 1
## [1] 2
## [1] 3
## [1] 4
## [1] 5
## [1] 6
## [1] 7
## [1] 8
## [1] 9
## [1] 10
## [1] 11

The control word next halts the processing of the current iteration step and advances the looping index.

i <- 0 
repeat{
  i <- i+1 
  if(i>=3 & i<=8) next 
  print(i)
  if (i>=10) break
  }
## [1] 1
## [1] 2
## [1] 9
## [1] 10

The while loop is stoped by a condition in the function call. Thus, in the while loop, a command block is iterated until the final condition is fulfilled.

i <- 0
while (i<=10) {
  i <- i + 1
}

The for loop iterates through all elements of a provided vector: for (i in M).

x <- c(3,6,2,5,10)
for (i in x) {
  print(i^2)
}
## [1] 9
## [1] 36
## [1] 4
## [1] 25
## [1] 100

Data import and export

R offers several options for accessing different data formats. Standard formats include text files, R files (single objects or entire workspaces), and databases. These data can be read in from files using a file path, but also from the Internet using an URL path or from the clipboard.

All objects created in R can be exported to file.

# data frames can be written to file by write.table and its variants
courseSchedule <- data.frame(weeks=1:10,
                             funFactor=sample(c('super','ok','pain'),10,replace=T))

write.table(courseSchedule,"courseSchedule.txt", row.names=F, col.names=T, sep=',')
# list all files in your folder
list.files()

# reading data frames
myDF <- read.table('courseSchedule.txt')
myDF <- read.table('courseSchedule.txt', sep=',', header=T)
Reading and writing R objects and scripts

It is crucial to record the way scripts were performed so we can reproduce them later. The usual workflow is to save your code in an R script (“.R file”).

You can go to “File/New File/R Script” to create a new R script. These files can be sourced in additional scripts to create clean pipelines. To effectively execute R expressions available in the script editor, use the most important keyboard shortcuts: Cmd/Ctrl + Enter.

Objects available in the environment can also be saved (“.RData file”).

# source external file
source("path-to-script.R")

# save objects to external file
myData <- c(1,2,3,4,5)
save(myData,file = "path-to-data.RData")

# save all objects in the environment to external file
save.image(myData,file = "path-to-data.RData")

# load objects from external file
load(file = "path-to-data.RData")
